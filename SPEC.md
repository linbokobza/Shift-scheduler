# מסמך אפיון — מערכת ניהול משמרות

---

## 1. תיאור כללי

אפליקציית Web לניהול סידורי עבודה שבועיים. המערכת מאפשרת לעובדים להגיש זמינות למשמרות, ולמנהלים ליצור, לערוך ולפרסם סידורי עבודה אוטומטית.

האפליקציה בנויה ב-RTL (ממשק בעברית), תומכת בדסקטופ ובמובייל, ומשתמשת ב-Linear Programming ליצירת סידור מאוזן ואופטימלי.

---

## 2. מטרות האפליקציה

- **לעובד:** הגשת זמינות שבועית למשמרות בצורה פשוטה ונוחה, וצפייה בסידור שפורסם.
- **למנהל:** צמצום הזמן שנדרש לבניית סידור עבודה — מהגשות הזמינות ועד לפרסום לעובדים — באמצעות אוטומציה.

---

## 3. קהל יעד

- **עובדים** בארגון שעובדים במשמרות (בוקר, ערב, לילה).
- **מנהלים** האחראים על בניית סידורי העבודה.

---

## 4. פיצ'רים מרכזיים

### עבור עובד

| פיצ'ר | תיאור |
|--------|--------|
| **הגשת זמינות** | בחירת זמינות (זמין / לא זמין) לכל משמרת בכל יום בשבוע, כולל הוספת הערה אישית לכל תא |
| **מועד הגשה** | הגבלת הגשה עד יום שלישי 12:00 של שבועיים לפני שבוע המשמרת, עם הצגת status ברור |
| **צפייה בסידור** | לאחר פרסום — צפייה בסידור המוגמר של השבוע |
| **לוח שנה** | צפייה בחופשות, ימי מחלה וחגים בתצוגת לוח שנה חודשי |
| **ניווט שבועי** | מעבר בין שבועות, עם כפתור מהיר לחזרה לשבוע ההגשה הנוכחי |

### עבור מנהל

| פיצ'ר | תיאור |
|--------|--------|
| **ניהול עובדים** | הוספה, הסרה, הפעלה/השבתה של עובדים, ואיפוס סיסמה |
| **ניהול חופשות ומחלות** | הוספת ימי חופשה/מחלה לעובדים לפי טווח תאריכים (לא כולל שישי-שבת) |
| **ניהול חגים** | הגדרת חגים מסוג "ללא עבודה" או "בוקר בלבד" |
| **יצירת סידור אוטומטי** | יצירת סידור שבועי מבוסס על כלל הנתונים, תוך שימוש ב-Linear Programming לאיזון משמרות |
| **עריכה ידנית** | שינוי שיוך עובד למשמרת, הוספת עובד נוסף למשמרת (extra assignment) |
| **נעילה והקפאה של תאים** | נעילת תא כדי שלא יינדד בסידור הבא; הקפאה כדי לנעול לצמיתות |
| **פרסום סידור** | שליחת הסידור לנראות עובדים |
| **תצוגת הגשות** | מעקב אחר כמה עובדים הגישו זמינות לשבוע הנבחר |
| **תצוגת זמינות מפורטת** | צפייה בכל הזמינויות לפי עובד ויום, עם אפשרות עריכה |
| **התרעות על משמרות** | הצגת אזהרות על משמרות ללא עובד זמין |
| **לוח שנה** | צפייה בחופשות וחגים בתצוגה חודשית |

---

## 5. זרימת משתמש (User Flow)

### עובד
```
כניסה → הגשת זמינות לשבוע הרלוונטי → שמירה → [לאחר פרסום] → צפייה בסידור
```

### מנהל
```
כניסה → מעקב אחר הגשות עובדים
       → יצירת סידור אוטומטי
       → עריכה ידנית (אם נדרש)
       → נעילת תאים חשובים
       → פרסום הסידור לעובדים
```
גם: ניהול עובדים / חופשות / חגים (בכל עת, ממסך הסייד-בר)

## 6. מסכים עיקריים

| מסך | תיאור |
|-----|--------|
| **Login** | כניסה עם אימייל וסיסמה; אפשרות שכחתי סיסמה |
| **דשבורד עובד** | טאב זמינות + טאב צפייה בסידור + לוח שנה |
| **דשבורד מנהל** | טבלת סידור שבועי + סייד-בר לניהול |
| **סייד-בר מנהל** | ניהול עובדים / חופשות / חגים |
| **מודל הגשות** | רשימת עובדים עם סטטוס הגשה לשבוע |
| **מודל שגיאות** | שגיאות ואזהרות שהתקבלו ביצירת הסידור |
| **לוח שנה** | תצוגה חודשית של חופשות, מחלות וחגים (משותף לעובד ולמנהל) |
| **דף ציבורי** | `/public` — סידור הנגיש ללא התחברות |
| **איפוס סיסמה** | `/reset-password` — שינוי סיסמה דרך לינק |

> כל הממשקים קיימים בשתי גרסאות: **דסקטופ** ו**מובייל** (tabs + pull-to-refresh).

## 7. דרישות טכניות

| קטגוריה | פרטים |
|----------|--------|
| **Frontend** | React 18, TypeScript, Vite, TailwindCSS |
| **State management** | TanStack React Query (server state), React useState (local) |
| **Routing** | React Router v7 |
| **HTTP** | Axios עם JWT auth בהדר |
| **Backend** | Node.js (תת-תיקייה `/backend`) |
| **אלגוריתם סידור** | LP Solver (`javascript-lp-solver`) לאופטימיזציה; fallback לאלגוריתם greedy |
| **גרפים** | Recharts |
| **ייצוא** | `html-to-image` לשמירת סידור כתמונה |
| **RTL** | ממשק מלא בעברית, כיוון RTL |
| **Offline support** | localStorage כ-cache מקומי |
| **Testing** | Vitest + Testing Library + MSW |
| **משמרות קבועות** | בוקר, ערב, לילה (ימי חול בלבד — שישי ערב/לילה ושבת אינם פעילים) |
| **מועד הגשה** | שלישי 12:00, שבועיים לפני שבוע המשמרת |
